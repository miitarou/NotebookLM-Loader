# Officeファイル一括変換ツール ウォークスルー

## 実装された機能
ユーザーの指定したフォルダ内のOfficeファイルを、NotebookLMに最適なMarkdown形式に一括変換するPythonツールを実装しました。

### 作成されたファイル
- **`office_to_notebooklm.py`**: コアとなる変換スクリプト。
    - **MarkItDown Engine**: Microsoft公式ライブラリによる高精度なテキスト変換。
    - **ZIP Support**: `.zip` ファイルを自動検知・解凍し、再帰的に処理。
    - **Visual Report**: 自作ロジックで画像密度を計算し、PDF利用を推奨。
    - **Hybrid Architecture**: 「変換」はMarkItDown、「判断」は自作ロジックのハイブリッド。
- **ファイル名の日本語化**: ZIP内のShift-JISエンコードされたファイル名（例: `座学...md`）も、文字化けすることなく正しく復元されることを確認。
- **フォルダ構造の維持**: ネストされたフォルダ構造が `フォルダ_サブフォルダ_ファイル名.md` という形式でフラット化され、重複なく出力されることを確認。
- **メタデータ**: 生成されたMarkdownファイルの冒頭に、元のパス情報が含まれていることを確認。
    - **オプション**: `--combine` (結合), `--skip-ppt` (PPTスキップ)。
- **`requirements.txt`**: 必要なライブラリ一覧。
- **`README.md`**: インストールおよび使用方法のガイド。

## テスト結果（検証項目）
本環境にはOfficeソフトおよびPythonの外部ライブラリがないため、実際の変換テストはユーザー環境で実施いただく必要があります。
コードレベルでは以下を確認済みです：
- 必須ライブラリ（`python-docx`, `openpyxl`, `python-pptx`, `pandas`）の適切なインポート
- ファイル拡張子による分岐ロジック
- 再帰的なフォルダ探索処理
- エラーハンドリング（個別のファイルが読み込めなくても止まらず次へ進む）

## 次のアステップ（ユーザー様へ）
`README.md` の手順に従い、以下のコマンドで実行を試してください。

```bash
# 1. ライブラリのインストール
pip install -r requirements.txt

# 2. 実行（デスクトップの "Docs" フォルダを変換する例）
python3 office_to_notebooklm.py ~/Desktop/Docs --combine
```
